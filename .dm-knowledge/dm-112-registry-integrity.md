- **[invariant]** A corrupt state/repos.json must never read as an empty registry. Every registry read goes through dm-lib's validated accessors (dm_registry_require_valid / dm_registry_has / dm_registry_keys / dm_registry_get / dm_merge_authority / dm_merge_allowed_bases); dm_ensure_dirs seeds only an absent or zero-length file, so a non-empty unparseable one is corruption and stops the operation. Never write a registry guard as 'jq -e ... && dm_die' — a jq parse error short-circuits the && and the guard silently does not fire. dm-doctor.sh is the one deliberate exception: it reads the registry raw because it DIAGNOSES corruption and must not refuse to run.  _(2026-07-21T04:48:35Z)_
- **[convention]** Recovery advice printed by the toolbelt must never name a destructive command unless the code has established the target is safe to destroy. dm-repo.sh's slot_obstruction_advice suggests rmdir only for a provably empty directory; a git repo or any non-empty path gets inspect-then-move-aside, because a blocked repo slot may be a live managed clone holding work that exists nowhere else (#112).  _(2026-07-21T04:48:39Z)_
- **[pitfall]** bash does not propagate set -e out of a command substitution nested inside a function that is itself called in a substitution (nor out of a case word, a [ ] argument, or a process substitution). A dm_die there prints and is discarded. So any script consuming a registry accessor must call dm_registry_require_valid in its MAIN shell — relying on a die inside $(dm_repo_dir ...) is not enough, and the degraded path composes to $DM_HOME, which passes a .git probe because the distro root is a git repo. tests/smoke.sh derives the must-validate script set from the tree so a new consumer is covered automatically.  _(2026-07-21T19:49:10Z)_
